# 产品需求文档：求职高手（Smart Pact）AI 求职辅助决策系统

**版本**: 1.0
**日期**: 2026-03-07

---

## 执行摘要

求职高手（Smart Pact）是一款面向高校应届毕业生的 AI 求职辅助决策系统，通过四个协同工作的 AI 智能体——鹰眼猎手、幻影投递官、深网调查员、契约卫士——覆盖从岗位发现到合同签署的完整求职流程。

当前应届毕业生在求职过程中面临招聘信息分散、企业信息不透明、劳动合同理解门槛高、投递流程繁琐等核心痛点。本系统通过 AI 技术将这些环节智能化、自动化，帮助缺乏求职经验的毕业生做出更明智的职业决策，有效规避求职风险。

本项目为学业项目（毕业设计），采用 Android 原生客户端 + Python 后端 + 商业 LLM API 的技术架构，MVP 阶段四大模块同步上线，数据范围限定于国内头部企业。

---

## 问题陈述

**当前痛点**：

1. **招聘信息分散**：应届生需要在多个平台之间反复切换查找岗位，效率低下且容易遗漏
2. **企业信息不透明**：缺乏经验的毕业生难以判断企业真实状况，容易踩坑（拖欠工资、劳动纠纷等）
3. **合同理解门槛高**：第一次签劳动合同，看不懂条款中的法律术语，可能签下不公平条款
4. **投递流程繁琐**：针对不同岗位定制简历、跟踪投递状态的工作量大且易出错

**解决方案**：

构建四智能体协同的 AI 系统，每个智能体专注解决一个核心痛点：
- 鹰眼猎手：聚合大厂招聘信息，基于用户画像智能匹配
- 幻影投递官：自动化简历定制与投递管理
- 深网调查员：多源数据构建企业风险画像
- 契约卫士：合同条款智能解析与风险标注

**预期成果**：
- 为毕业生提供一站式求职辅助工具，减少信息不对称
- 通过 AI 能力降低求职各环节的认知门槛
- 作为毕业设计，展示多智能体协作、RAG、数据聚合等技术能力

## 用户画像

### 主要用户：应届毕业生小林

- **角色**：计算机相关专业大四学生
- **目标**：找到一份匹配自身能力的大厂技术岗位，避免踩坑
- **痛点**：
  - 第一次求职，不了解行业和企业的真实情况
  - 不知道如何针对不同岗位调整简历
  - 看不懂劳动合同中的法律条款
  - 在多个招聘渠道间疲于奔命
- **技术水平**：中高级（计算机专业学生，熟练使用智能手机）
- **使用场景**：秋招/春招季，每天花 1-2 小时使用系统辅助求职

## 用户故事与验收标准

### 故事 1：智能岗位发现与匹配

**作为** 应届毕业生
**我想要** 系统自动帮我从大厂官网收集适合我的岗位信息
**以便于** 不用逐个网站去搜索，快速找到最匹配的机会

**验收标准：**
- [ ] 系统能够定时爬取预配置的国内大厂招聘官网（如华为、腾讯、阿里等）
- [ ] 用户填写求职画像后，系统基于画像对岗位进行匹配度评分
- [ ] 岗位列表支持按匹配度、发布时间、公司等维度排序和筛选
- [ ] 能识别并标注异常岗位（如长期未更新、要求与薪资不匹配等）

### 故事 2：企业风险调查

**作为** 应届毕业生
**我想要** 在投递前了解目标企业的真实经营状况和风险信息
**以便于** 避免入职问题企业

**验收标准：**
- [ ] 输入企业名称后，系统调用天眼查等 API 获取企业工商、司法、经营数据
- [ ] 生成企业风险画像，包含司法风险、经营风险、舆情风险等维度
- [ ] 每个风险维度有明确的风险等级标注（高/中/低）
- [ ] AI 生成通俗易懂的风险评估报告摘要

### 故事 3：简历定制与投递管理

**作为** 应届毕业生
**我想要** 系统根据不同岗位要求帮我优化简历，并统一管理投递进度
**以便于** 提高投递效率和简历通过率

**验收标准：**
- [ ] 支持上传简历文件（PDF/Word），系统解析简历内容
- [ ] 针对特定岗位 JD，AI 生成简历优化建议
- [ ] 提供投递记录管理，追踪各岗位的投递状态（待投递/已投递/笔试/面试/offer）
- [ ] 投递状态变更时提供提醒通知

### 故事 4：劳动合同风险审查

**作为** 应届毕业生
**我想要** 上传劳动合同后，系统帮我逐条分析条款的合理性和风险
**以便于** 签约前了解自己的权益，避免签下不公平条款

**验收标准：**
- [ ] 支持上传合同文件（PDF/Word/图片），图片支持 OCR 识别
- [ ] 系统对合同进行结构化解析，提取关键条款（薪资、试用期、竞业协议等）
- [ ] 每条关键条款标注风险等级，并提供通俗解读
- [ ] 与劳动法相关规定进行对标，标注不合规或不利条款
- [ ] 生成整体风险摘要和签约建议

### 故事 5：AI 对话助手

**作为** 应届毕业生
**我想要** 通过自然语言对话获取求职建议和操作各模块功能
**以便于** 以最自然的方式使用系统

**验收标准：**
- [ ] 全局 AI 对话入口，支持自然语言交互
- [ ] 能够识别用户意图并分发到对应智能体模块
- [ ] 支持多轮对话上下文管理
- [ ] 支持流式输出（打字机效果）
- [ ] 可回答通用求职问题（面试技巧、行业分析等）

## 功能需求

### 模块一：鹰眼猎手（Eagle Hunter）

**功能描述**：持续扫描国内大厂招聘官网，基于用户画像进行多维度岗位匹配分析。

**核心功能：**
1. **岗位信息采集**
   - 定时爬取预配置的大厂招聘官网（MVP 阶段 10-15 家头部企业）
   - 结构化提取岗位名称、要求、薪资范围、工作地点等信息
   - 增量更新，避免重复抓取

2. **用户画像管理**
   - 用户填写个人信息：专业、技能、意向城市、期望薪资等
   - 画像信息作为匹配算法的输入

3. **智能匹配与推荐**
   - 基于用户画像与岗位 JD 的语义匹配（调用 LLM）
   - 匹配度评分与排序
   - 异常岗位识别与预警

**用户流程**：
用户设置画像 → 系统后台采集岗位 → 匹配推荐 → 用户浏览/筛选 → 查看详情 → 跳转调查/投递

### 模块二：幻影投递官（Phantom Delivery）

**功能描述**：解析用户简历，提供 AI 驱动的简历优化建议，管理投递全流程。

**核心功能：**
1. **简历解析**
   - 支持 PDF、Word 格式上传
   - 结构化提取简历信息（教育背景、项目经验、技能等）

2. **AI 简历优化**
   - 针对目标岗位 JD，LLM 生成简历优化建议
   - 建议覆盖内容措辞、关键词匹配、结构调整等

3. **投递管理**
   - 投递记录的增删改查
   - 状态流转管理（待投递 → 已投递 → 笔试 → 面试 → offer/拒绝）
   - 投递统计与数据看板

**用户流程**：
上传简历 → 选择目标岗位 → 获取优化建议 → 记录投递 → 追踪状态

### 模块三：深网调查员（Deep Net Investigator）

**功能描述**：整合企业工商、司法、舆情等多源公开数据，构建企业风险画像。

**核心功能：**
1. **企业数据采集**
   - 接入天眼查/企查查等企业信息 API
   - 获取工商信息、司法记录、经营异常、行政处罚等数据

2. **风险画像构建**
   - 司法风险：诉讼记录、失信记录、被执行信息
   - 经营风险：经营异常、行政处罚、税务问题
   - 舆情风险：负面新闻（裁员、欠薪、劳动纠纷等）

3. **AI 风险评估**
   - LLM 综合分析各维度数据，生成风险等级评定
   - 生成通俗易懂的风险评估报告
   - 给出求职建议（建议/谨慎/避免）

**用户流程**：
输入企业名称 → 系统调用 API 获取数据 → AI 分析 → 展示风险画像 → 查看详细报告

### 模块四：契约卫士（Contract Guardian）

**功能描述**：对劳动合同进行结构化解析，提取关键条款，标注风险点并提供通俗解读。

**核心功能：**
1. **文件处理**
   - 支持 PDF、Word 格式合同上传
   - 支持图片上传 + OCR 文字识别
   - 文本提取与预处理

2. **条款解析**
   - 结构化提取关键条款：薪资待遇、试用期、工作时间、竞业协议、违约金等
   - 识别合同类型（劳动合同、实习协议、劳务合同等）

3. **风险分析**
   - 逐条款风险评估与等级标注
   - 与《劳动合同法》等法规对标合规性检查
   - 通俗语言解读法律术语
   - 整体风险摘要与签约建议

**用户流程**：
上传合同 → 系统解析文本 → AI 逐条分析 → 展示风险标注 → 查看详细解读 → 获取签约建议

### 模块五：统一 AI 对话服务

**功能描述**：全局 AI 对话入口，支持自然语言交互，智能分发至各模块。

**核心功能：**
1. **意图识别与分发**
   - 识别用户对话意图，路由到对应智能体
   - 支持快捷指令（如 "/查公司 腾讯"、"/分析合同"）

2. **多轮对话管理**
   - 上下文感知的连续对话
   - 流式输出（SSE）实现打字机效果

3. **通用求职问答**
   - 回答面试技巧、行业分析等通用问题
   - 模拟面试练习

### 不在范围内（Out of Scope）

- 第三方招聘平台数据爬取（Boss 直聘、拉勾等）
- 自动代替用户投递简历
- 社交功能（求职社区、经验分享）
- iOS 客户端
- Web 客户端
- 支付/会员/商业化功能
- 用户间通讯功能

## 技术约束

### 系统架构

```
Android 客户端（Jetpack Compose）
        ↕ RESTful API + SSE
Python 后端（FastAPI）
        ↕
  ┌─────┼─────┐
  ↓     ↓     ↓
PostgreSQL  商业LLM API  第三方API
(数据存储)  (AI能力)    (天眼查等)
```

### 前端技术栈
- **框架**：Jetpack Compose（声明式 UI）
- **语言**：Kotlin
- **设计系统**：Material 3
- **最低 SDK**：API 26（Android 8.0）
- **目标 SDK**：API 36（Android 15）

### 后端技术栈
- **框架**：FastAPI（Python）
- **数据库**：PostgreSQL
- **ORM**：SQLAlchemy / Tortoise ORM
- **任务队列**：Celery + Redis（定时爬虫任务）
- **文件存储**：对象存储（阿里云 OSS / 腾讯云 COS）

### AI 能力
- **LLM**：商业 API（OpenAI / Claude / 国内大模型 API）
- **应用场景**：
  - 岗位匹配语义分析
  - 简历优化建议生成
  - 企业风险报告生成
  - 合同条款解析与风险评估
  - 对话意图识别与回答

### 第三方集成
- **天眼查/企查查 API**：企业工商、司法、经营数据
- **OCR 服务**：合同图片文字识别（百度 OCR / 腾讯 OCR）
- **大厂招聘官网**：定时爬取岗位信息（遵守 robots.txt）

### 性能要求
- API 响应时间 < 2s（非 AI 接口）
- AI 生成类接口采用流式输出，首 token < 3s
- 岗位爬取任务：每日定时执行 1-2 次
- 并发用户支持：50+（学业项目规模）

### 安全要求
- 用户认证：基础账号密码 + JWT Token
- 通信加密：HTTPS
- 文件上传：类型和大小校验，防止恶意文件
- API 鉴权：所有接口需携带有效 Token
- 敏感数据（简历、合同）服务端存储，不在客户端缓存

### 部署
- **平台**：国内云服务（阿里云 / 腾讯云）
- **方式**：Docker 容器化部署
- **环境**：开发环境 + 演示环境

## MVP 范围与分阶段计划

### Phase 1：MVP（核心功能）

四大模块同步上线，每个模块实现核心链路：

**鹰眼猎手 MVP：**
- 爬取 10-15 家头部大厂招聘官网
- 用户画像填写
- 基础岗位匹配与推荐列表

**幻影投递官 MVP：**
- 简历上传与解析
- AI 简历优化建议（基于单个岗位 JD）
- 投递记录管理（手动录入状态）

**深网调查员 MVP：**
- 企业名称搜索
- 基础工商信息 + 司法风险展示
- AI 生成风险摘要

**契约卫士 MVP：**
- PDF/Word 合同上传与解析
- 关键条款提取与风险标注
- AI 生成通俗解读

**统一 AI 对话 MVP：**
- 基础意图识别（4 模块 + 通用问答）
- 单轮/简单多轮对话
- 流式输出

### Phase 2：功能完善（如时间允许）

- 岗位变动推送通知
- 简历版本管理（多版本对比）
- 企业舆情监控（新闻爬取）
- 合同 OCR 图片识别
- 模拟面试对话练习
- 数据统计看板

### 未来展望

- 更多企业招聘源接入
- 端侧轻量模型推理（降低 API 调用成本）
- RAG 检索增强（劳动法知识库）
- 求职社区与经验分享

## 风险评估

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|----------|
| 大厂官网反爬机制 | 高 | 高 | 控制爬取频率，遵守 robots.txt，做好数据缓存，预备人工录入方案 |
| LLM API 调用成本 | 中 | 中 | 使用国产模型 API（成本较低），对非实时任务做结果缓存 |
| 天眼查 API 调用配额限制 | 中 | 中 | 申请开发者额度，做好本地数据缓存，限制用户查询频率 |
| 合同解析准确度 | 中 | 高 | 使用高能力 LLM，提供 "仅供参考" 免责提示，关键条款人工标注验证 |
| OCR 识别准确度 | 中 | 中 | MVP 阶段优先支持 PDF/Word，图片 OCR 作为 Phase 2 |
| 开发周期紧张 | 高 | 高 | 各模块先完成核心链路，非核心功能后置；善用 AI 辅助开发 |

## 术语表

| 术语 | 定义 |
|------|------|
| 智能体（Agent） | 专注于特定任务的 AI 模块，具备独立的数据处理和 AI 分析能力 |
| JD（Job Description） | 岗位描述，包含岗位职责、任职要求等信息 |
| 用户画像 | 用户填写的个人求职偏好信息，用于岗位匹配 |
| 风险画像 | 系统基于多源数据为企业生成的风险评估报告 |
| RAG | 检索增强生成，结合知识库检索与 LLM 生成的技术 |
| OCR | 光学字符识别，将图片中的文字转换为可编辑文本 |
| SSE | Server-Sent Events，服务端推送技术，用于实现流式输出 |
| MVP | 最小可行产品，包含核心功能的首个可用版本 |
